Connect a Claude client

Point Claude at this MCP server with one of the gateway tokens above. In each example, replace your-mcp-host with wherever the MCP server (hr-mcp) is reachable — default port 8100, path /mcp — and swap <paste-a-gateway-token> for a token you generated. Click a block to select it.

Claude Desktop — add to claude_desktop_config.json (Settings → Developer → Edit Config), then restart. Requires Node.js.
{% raw %}
{
  "mcpServers": {
    "hr-sot": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://your-mcp-host:8100/mcp",
        "--header",
        "Authorization:${HRSOT_MCP_TOKEN}"
      ],
      "env": {
        "HRSOT_MCP_TOKEN": "Bearer <paste-a-gateway-token>"
      }
    }
  }
}
{% endraw %}

The token lives in env (spaces are fine there) while the header uses Authorization:${HRSOT_MCP_TOKEN} with no space — a documented workaround for how npx mangles spaces inside args on Claude Desktop/Windows.

Claude Code — one command:
{% raw %}
claude mcp add --transport http hr-sot https://your-mcp-host:8100/mcp \
  --header "Authorization: Bearer <paste-a-gateway-token>"
{% endraw %}
…or add to .mcp.json (project) / ~/.claude.json:
{% raw %}
{
  "mcpServers": {
    "hr-sot": {
      "type": "http",
      "url": "https://your-mcp-host:8100/mcp",
      "headers": {
        "Authorization": "Bearer <paste-a-gateway-token>"
      }
    }
  }
}
{% endraw %}
