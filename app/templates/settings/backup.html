Backup & Restore

Export the whole instance to a zip, or restore it from one.

Export backup

Downloads a .zip containing the database and this instance's secret keys (session, JWT, and provider secret). Because it includes secrets, store the file somewhere safe. Set a password to AES-256 encrypt the archive.

If set, you'll need this password to restore the backup.

⚠ Restore (replaces all data)

Uploading a backup overwrites the current database and secret keys with the file's contents. This cannot be undone. After a restore you may need to sign in again, and a full app restart is recommended so the restored session key takes effect.
